## ✅ 코드 수정 시 필수 체크리스트

1. 새로운 함수나 변수 추가 시 필요한 import 문 반드시 먼저 확인하고 추가.
2. datetime 사용 시 → from datetime import datetime 확인
3. 수정 후 해당 파일의 import 섹션을 반드시 검토.
4. 문법 에러가 날 가능성이 있는 코드는 수정 후 반드시 `python3 -c "import ast; ast.parse(open('파일명').read())"` 로 문법 체크.
5. 같은 실수 반복 금지. 이전에 고친 내용을 또 깨뜨리지 말 것.
6. **코드 수정 후 반드시 python3 -c로 문법 검증 실행. 검증 실패 시 수정 완료 보고 금지.**
7. **파일을 수정한 후 반드시 git add → git commit → git push 까지 실행. commit 없이 수정 완료 보고 금지.**

## 🔴 절대 실수 금지 규칙

1. LINE 메시지에 마크다운(**볼드**, *이탤릭*) 절대 사용 금지. 섹션 구분은 【】만 사용.
2. 시간 표현: 24시간 표기(15時 등) 금지. 午後、夕方、夜 등 범위만 사용.
3. 사주 명식: 입력된 saju 데이터만 사용. 창작 절대 금지.
4. 不明 입력 시 시주(時柱) 언급 절대 금지.
5. prescription 모드: preview 내용 반복 금지. 액션만.
6. 직업 단정 표현 금지: 仕事→今あなたが最もエネルギーを注いでいること
7. 현재 시간 이후로만 조언. 이미 지난 시간대 조언 금지.
8. 鑑定予約 인식 시 따옴표/특수문자 포함 입력도 인식.
9. 유료 처방전(prescription)은 무료(preview) 내용 복사 금지. 더 깊게 작성.

## ⚠️ 자주 겪는 문제 & 대응법

### 문제 1: 코드가 갑자기 꼬임
**원인**: 한 번에 너무 많이 수정하거나, 계획 없이 바로 구현할 때 발생
**대응**:
- 작업 전 반드시 플랜 먼저 요청
- 꼬이면 이전 상태로 되돌리고 작은 단위로 다시 시작
- 프롬프트: `"지금 코드가 꼬였어. 원인 먼저 분석해줘. 수정하지 말고."`

### 문제 2: 같은 실수 반복
**원인**: 세션마다 Claude 기억이 초기화됨
**대응**:
- 실수 발생 즉시 위 "실수 기록" 섹션에 추가
- 프롬프트: `"이 실수를 CLAUDE.md 실수 기록에 추가해줘"`
- 세션 시작 시 항상 CLAUDE.md 읽게 하기

### 문제 3: 세션 바뀌면 처음부터 설명해야 함
**원인**: Claude는 대화가 끊기면 이전 내용을 전혀 모름
**대응**:
- 세션 시작마다: `"CLAUDE.md 읽고 현재 프로젝트 구조 파악한 다음 시작해"`
- 중요한 결정/구조는 항상 CLAUDE.md에 기록해두기
